# Smart User Data Cleaner v1  
## Week 1 – Day 1: Data Architecture & Analytical Thinking Foundations

### Overview
Day 1 focused on building the foundational mindset required for Data Science by understanding how data is structured, organized, and processed before analysis. Rather than immediately jumping into coding, the goal was to develop analytical thinking and understand the architecture behind data systems.

---

## Main Objectives
- Understand what data architecture means
- Learn the difference between raw data and structured data
- Recognize how messy real-world data can impact analysis
- Develop problem-solving logic before coding
- Build the foundation for future data preprocessing projects

---

## Key Concepts Learned

### 1. What is Data Architecture?
Data architecture refers to how data is collected, stored, organized, managed, and transformed for efficient use.

### 2. Structured vs Unstructured Data
**Structured Data:**  
Organized, formatted, and easily searchable data (e.g., tables, CSV files, SQL databases)

**Unstructured Data:**  
Messy or inconsistent data without a predefined format (e.g., raw user text, emails, social media posts)
